HTML

HTML (HyperText Markup Language) jest fundamentalnym językiem używanym do tworzenia stron internetowych. Składa się z zestawu tagów, które definiują strukturę dokumentu. Na przykład, zawiera metadane takie jak tytuł strony i linki do arkuszy stylów, podczas gdy zawiera główną treść widoczną dla użytkownika. Poprzez odpowiednie użycie tagów, HTML pozwala na tworzenie czytelnych i semantycznie poprawnych stron, co ułatwia zarówno ich tworzenie, jak i indeksowanie przez wyszukiwarki internetowe.

Kod HTML składa się z elementów, które mogą być zagnieżdżone wewnątrz siebie, tworząc hierarchiczną strukturę dokumentu. Na przykład, <div> może zawierać inne elementy, takie jak tekst, obrazy lub formularze, a <span> może definiować fragmenty tekstu, które mają być stylizowane w sposób szczególny. Przy użyciu różnych atrybutów, takich jak "class" lub "id", programiści mogą dostosowywać wygląd i zachowanie elementów strony, co umożliwia tworzenie atrakcyjnych i interaktywnych witryn internetowych. 

        x = 5;
y = 6;
z = x + y;

CSS

CSS (Cascading Style Sheets) to język używany do stylizacji stron internetowych. Jest wykorzystywany do kontrolowania wyglądu i układu elementów HTML, co pozwala na nadanie stronie odpowiedniego wyglądu i formatowania. CSS definiuje różne właściwości, takie jak kolor, rozmiar czcionki, marginesy, czy położenie elementów na stronie. Dzięki zewnętrznym arkuszom stylów (plikom .css) można łatwo zarządzać wyglądem wielu stron internetowych, co ułatwia utrzymanie spójności wizualnej oraz zmniejsza powtarzalność kodu. 

        border-left: 10px double rgba(0, 0, 0, 0.75);

Kaskadowość w CSS oznacza, że style mogą być dziedziczone przez elementy podrzędne, co pozwala na efektywne zarządzanie wyglądem witryny. Dodatkowo, kaskadowość umożliwia nadpisywanie stylów, co daje możliwość dostosowania wyglądu poszczególnych elementów bez konieczności zmiany całej struktury. W ten sposób, CSS zapewnia elastyczność i kontrolę nad prezentacją stron internetowych, co jest kluczowe dla tworzenia atrakcyjnych i funkcjonalnych interfejsów użytkownika. 

        background: linear-gradient(gradientDirection, color1, color2, ...);
border-left: 10px double rgba(0, 0, 0, 0.75);

JavaScript

JavaScript jest wszechstronnym językiem programowania stosowanym głównie do tworzenia interaktywnych stron internetowych. Jest on interpretowany przez przeglądarkę internetową, co umożliwia dynamiczną manipulację zawartością strony oraz reakcję na interakcje użytkownika. JavaScript może być również używany po stronie serwera (Node.js) do tworzenia aplikacji internetowych i serwerów.

Dodatkowo, JavaScript posiada obszerną społeczność i bogatą bibliotekę frameworków, takich jak React, Angular czy Vue.js, które przyspieszają i ułatwiają rozwój aplikacji internetowych.

React to popularna biblioteka JavaScript, która umożliwia tworzenie interfejsów użytkownika (UI) w sposób modułowy i efektywny. Jest ona rozwijana i utrzymywana przez Facebooka oraz społeczność programistów, co sprawia, że jest stale aktualizowana i rozwijana. Jedną z głównych zalet Reacta jest jego podejście oparte na komponentach, które pozwalają na tworzenie reużywalnych części interfejsu. Komponenty te są niezależne, co ułatwia zarządzanie nimi i wprowadzanie zmian w interfejsie.

React wykorzystuje tzw. Virtual DOM, czyli wirtualne drzewo DOM, które pozwala na minimalizację operacji na rzeczywistym drzewie DOM. Dzięki temu React jest wydajny i szybki, co przekłada się na płynne działanie aplikacji nawet przy dużej ilości danych i interakcji użytkownika. Ponadto, React promuje jednokierunkowy przepływ danych (one-way data binding), co sprawia, że zarządzanie stanem aplikacji jest czytelne i przewidywalne. To wszystko sprawia, że React jest popularnym wyborem dla twórców nowoczesnych aplikacji internetowych, zapewniając im narzędzia do szybkiego i efektywnego tworzenia interfejsów użytkownika. 

        function HelloWorld()

Backend

Backend, zwany również warstwą serwerową, odgrywa kluczową rolę w działaniu aplikacji internetowych. Jest to część systemu, która odpowiada za przetwarzanie danych, zarządzanie bazami danych, autoryzację użytkowników oraz obsługę logiki biznesowej. Backend komunikuje się z frontendem (interfejsem użytkownika) poprzez protokoły sieciowe, takie jak HTTP, przekazując żądania klienta do odpowiednich komponentów aplikacji.

Backend często składa się z wielu komponentów, takich jak serwery aplikacji, bazy danych, serwery plików oraz systemy cache. Serwery aplikacji, takie jak Node.js, Python Django, Ruby on Rails czy Java Spring, są odpowiedzialne za przetwarzanie żądań klienta oraz generowanie odpowiedzi. Bazy danych, takie jak MySQL, PostgreSQL czy MongoDB, przechowują dane potrzebne do działania aplikacji, umożliwiając ich bezpieczne i skuteczne zarządzanie. Serwery plików obsługują przesyłanie plików, takich jak obrazy czy dokumenty, między klientem a serwerem, natomiast systemy cache zapewniają szybki dostęp do często używanych danych, przyspieszając działanie aplikacji.
